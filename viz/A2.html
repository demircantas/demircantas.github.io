Subtheme: [Transit Oriented Development]

Overall Analysis Questions

  1. How does the frequency and connectivity of MBTA services at different stations correlate with the surrounding population density?
  2. What is the relationship between the proximity MBTA stations and the availability of amenities such as parks, schools, and retail establishments?
  3. How do property values vary in relation to their distance from MBTA stations?

    4. My motivation for these questions stems from a desire to understand how transit infrastructure can shape urban dynamics and improve quality of life. By focusing on the MBTA network, I aim to explore how transit connectivity (node value) relates to population density, how proximity to transit enhances urban amenities (place value), and how these factors influence property values (market value). This inquiry is inspired by the comprehensive 3V approach, which emphasizes the interconnected benefits of effective transit-oriented development as described in the World Bank article Transforming the Urban Space Through Transit-Oriented Development: The 3V Approach. Analyzing these aspects will not only shed light on the current state of urban development in transit-accessible areas but also provide insights into potential strategies for sustainable growth.

Dataset Name Description Topics Covered
mbta_lines.geojson This dataset, derived from the MBTA's GTFS feed, represents transit routes as multiline geometries in GeoJSON format. Each feature corresponds to a transit route operated by the MBTA, with attributes such as route_id, route_long_name, route_color, and route_url. The dataset is converted from routes.txt into a MULTILINESTRING format using gtfs-to-geojson, with a deduplicated version removing redundant route_long_name entries. Used for transit network visualization, route mapping, and accessibility analysis. Beneficial for urban planning, public transportation studies, and integrating with real-time vehicle tracking.
mbta_lines_with_buffer.geojson This dataset extends mbta_lines.geojson by incorporating a 50-meter buffer around transit routes, representing them as multipolygon geometries in GeoJSON format. Extracted from routes.txt in the GTFS feed, it includes attributes like route_id, route_long_name, route_color, and route_url. The geometry is stored as MULTIPOLYGON, offering a broader spatial representation of MBTA transit corridors. Useful for transit impact assessments, accessibility studies, and urban infrastructure planning. Supports analysis of pedestrian and cyclist access to transit routes and helps visualize the influence of transit networks on surrounding areas.
mbta_stops.geojson This dataset contains all transit stops in the MBTA network, extracted from the GTFS stops.txt file and enriched with route associations. Each stop is represented as a GeoJSON Point with attributes such as stop_name, stop_id, zone_id, wheelchair_boarding, and routes. The dataset provides detailed information about MBTA stops, including their locations and accessibility status. Useful for transit stop mapping, accessibility analysis, and multimodal transportation planning. It supports applications in public transit optimization, urban mobility studies, and stop clustering analysis.
mbta_stops_collapsed.geojson This dataset is a deduplicated version of mbta_stops.geojson, where stops with identical stop_name, zone_id, and stop_url have been collapsed to a single entry. Through this process, 569 duplicate stops were removed, improving data clarity while retaining essential stop-level attributes. Helps in reducing redundancy in transit stop datasets, improving efficiency for GIS applications and urban mobility research. Suitable for clustering analyses, transit station accessibility studies, and ridership estimations.
mbta_stops_with_buffer_collapsed.geojson This dataset extends mbta_stops_collapsed.geojson by adding a buffer around each stop, representing a pedestrian-accessible area. Each stop is stored as a GeoJSON Polygon instead of a Point, reflecting a defined spatial reach around the stop for transit accessibility studies. Valuable for walkability assessments, transit coverage analysis, and studies on first/last-mile connectivity. Supports urban planning by visualizing how well transit stops serve surrounding areas.
mbta_stops_with_buffer.geojson This dataset was generated from the MBTA's GTFS feed, adding a 0.5 mile (805 meters) buffer around each transit stop, creating polygonal representations for the stops. The data originally extracted from stops.txt file includes stop details such as stop_name, zone_id, and routes, all transformed into a GeoJSON format with polygons representing buffered areas. Essential for accessibility studies, urban planning, and the creation of walkable areas around transit stops. It helps visualize the potential influence of transit accessibility on urban development and planning.
mbta_stops_with_buffer_collapsed.geojson Derived from mbta_stops_with_buffer.geojson, this dataset removes duplicate entries for stops sharing the same stop_name, zone_id, and stop_url, simplifying the data set while maintaining a spatial buffer around each unique stop as a GeoJSON polygon. Useful in minimizing data redundancy in urban mobility research, optimizing transit-oriented development studies, and supporting policy-making for enhanced transit stop accessibility and connectivity.
